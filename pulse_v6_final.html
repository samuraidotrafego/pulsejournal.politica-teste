1
Criar conta na Pepper
Tempo: 30 minutos · pepper.com.br
A Pepper é onde o dinheiro entra. É o primeiro passo porque tudo gira em torno dela — quando alguém paga na Pepper, ela avisa o Make, que avisa o Z-API, que manda a mensagem de boas-vindas no WhatsApp.
O que fazer:
1. Acesse pepper.com.br e clique em "Criar conta"
2. Preencha seus dados e valide o e-mail
3. Clique em "Novo Produto" → "Assinatura Recorrente"
4. Crie o plano "Pro Mensal" com valor R$47,00 e trial de 72 horas
5. Copie o link de checkout que a Pepper gerar — você vai precisar dele na landing page
2
Conectar o Z-API
Tempo: 20 minutos · z-api.io · R$97/mês
O Z-API é a ponte entre o seu WhatsApp Business e o sistema. Sem ele, você precisaria enviar cada mensagem manualmente. Com ele, o sistema envia sozinho — boas-vindas, briefing das 7h, breaking news — tudo automático.
O que fazer:
1. Crie um número de WhatsApp Business SEPARADO do seu pessoal
2. Acesse z-api.io e crie uma conta
3. Clique em "Nova Instância" e escaneie o QR code com o WhatsApp Business
4. Copie o "Instance ID" e o "Token" — você vai usar no Make
5. Teste enviando uma mensagem de teste para seu número pessoal via Z-API
3
Configurar o Supabase
Tempo: 30 minutos · supabase.com · Grátis
O Supabase é o banco de dados — onde ficam armazenados todos os seus assinantes, notícias e dados. Pense nele como uma planilha do Excel super avançada que qualquer sistema pode acessar automaticamente.
O que fazer:
1. Acesse supabase.com → "Start your project" → "New project"
2. Nome: "pulse-journal" · Região: São Paulo · Anote a senha
3. Vá em "Table Editor" → "New Table" → crie a tabela "subscribers"
4. Adicione as colunas: id (int8), name (text), phone (text), email (text), plan (text), status (text), created_at (timestamp)
5. Repita para as tabelas "news" e "quiz_results"
6. Vá em "Settings" → "API" → copie a URL e a anon key
4
Criar os cenários no Make
Tempo: 2 horas · make.com · Grátis início
O Make é onde você conecta tudo sem escrever uma linha de código. Você arrasta blocos visuais e define o que acontece quando algo ocorre. É a cola entre todas as ferramentas.
Cenário 1 — Pagamento (o mais importante):
Bloco 1: "Webhooks" → "Custom Webhook" (copie a URL e cole na Pepper)
Bloco 2: "Supabase" → "Create a Record" → tabela subscribers
Bloco 3: "HTTP" → chame a Z-API com a mensagem de boas-vindas
Bloco 4: "Beehiiv" → "Add Subscriber"
Ligue os blocos com setas. Clique em "Run once" para testar.
Cenário 2 — Briefing diário (cron às 6h50):
Bloco 1: "Schedule" → "Every day" → 6h50
Bloco 2: "Supabase" → buscar notícias com status "approved"
Bloco 3: "HTTP" → Z-API dispara para cada assinante ativo
5
Construir o pipeline editorial com Claude Code
Tempo: 2-3 horas · terminal do computador
Aqui é onde o Claude Code entra. Você abre o terminal, digita o que quer em português, e ele constrói o código. Você não precisa entender o código — só precisa testar se funciona.

O maior perigo não é o mercado rejeitar a ideia. É você se perder na complexidade do produto antes de ter o primeiro assinante pago. O MVP mínimo primeiro — sempre.
Risco 1 — Financeiro
Meta de R$20K em 30 dias é agressiva demais
A meta depende de 3 canais funcionando ao mesmo tempo desde o dia 1: lista fria convertendo, conteúdo orgânico crescendo e produto técnico sem erros. Se qualquer um travar, a meta trava junto. Estimativa realista: R$5K a R$12K no mês 1 se executar bem. R$15K a R$20K entre o mês 2 e o mês 3.
Atacar a lista de 1M de empreendedores ANTES de qualquer ad pago. A lista é seu maior ativo — usá-la primeiro elimina o custo de aquisição da primeira onda. Com conversão de 0,03% (3 em cada 10.000), você já tem 300 assinantes = R$14.100. Ads são a segunda onda.
Risco 2 — Técnico
Produto complexo demais para construir sozinho em 7 dias
O MVP que desenhamos tem Painel + NEXUS + quiz + automações + landing + blog. Isso não é 7 dias — são 3 semanas. Se você tentar construir tudo de uma vez, vai entregar nada.
3 MVPs em sequência, não um MVP gigante. MVP1 (dias 1-5): só WhatsApp funcionando — já cobra. MVP2 (dias 6-9): quiz + landing. MVP3 (dias 10-14): Painel com Polymarket. Se o MVP3 atrasar, você já tem receita rodando — não é uma crise.
Risco 3 — Editorial
Neutralidade política é difícil de manter sob pressão
O público político brasileiro tende a desconfiar de quem diz ser neutro. A direita vai achar que você é de esquerda. A esquerda vai achar o contrário. Você vai receber pressão para tomar lado. Se ceder uma vez, perde a credibilidade central do produto.
Você já respondeu: as notícias falam por si. O prompt fixo do NEXUS e da Claude API não tem opinião — tem fato + as 3 linhas de consequência. Você não declara neutralidade — executa neutralidade. Isso é impossível de refutar.
Risco 4 — Timing
Produto imaturo no pico de outubro
Se o produto não tiver audiência consolidada até agosto, outubro vai passar rápido demais para aproveitar. Agosto é quando os debates começam e o interesse político vira mainstream. Quem chega em setembro está chegando tarde.
Parar de pensar em outubro como destino e começar a pensar em agosto como prazo real. Isso significa que os 14 dias não são sobre vender 426 assinantes — são sobre construir a fundação que vai escalar até agosto. Cada semana de atraso custa posicionamento no pico.
Veredicto honesto
É um projeto bom com timing real.
O modelo de negócio faz sentido. O nicho existe e está virgem. A Not Journal validou o modelo (newsletter + terminal + WhatsApp) no Brasil. O Polymarket validou o interesse em dados eleitorais. Você quer cruzar as duas coisas — isso faz sentido como tese de produto.

O risco principal não é a ideia — é a execução. Disciplina no MVP mínimo primeiro. Consistência no conteúdo diário sem falhar. Firmeza na neutralidade mesmo quando der pressão. Se fizer esses três, o produto tem condição real de escalar até outubro de 2026 com audiência significativa.

Cada termo
em português simples
Seu primeiro projeto SaaS com IA. Você não precisa decorar — mas precisa entender o que cada coisa significa para tomar boas decisões.
SaaS
Modelo de negócio
Software as a Service. Pessoas pagam mensalmente para usar um produto digital. O PULSE.JOURNAL é SaaS: cobra todo mês, o assinante acessa enquanto paga. Para de pagar, perde o acesso.Analogia: é como a Netflix. Você não compra os filmes — paga pelo acesso todo mês.
MRR
Métrica financeira
Monthly Recurring Revenue — Receita Recorrente Mensal. 426 assinantes × R$47 = MRR de R$20.022. A métrica mais importante de um SaaS — mostra se está crescendo ou afundando.Analogia: é como o salário do negócio — o valor que entra todo mês independente de novas vendas.
Churn
Métrica
Taxa de cancelamento. 100 assinantes, 5 cancelam = churn de 5%. É o maior inimigo de um SaaS. O Painel e o breaking news existem para criar hábito e reduzir o churn.Analogia: balde furado. Você enche de água (novos assinantes), mas se o buraco for grande, nunca fica cheio.
Webhook
Tecnologia
Notificação automática entre sistemas. Quando alguém paga na Pepper, ela avisa o Make automaticamente. Esse aviso é o webhook — o Make recebe e executa as ações seguintes.Analogia: carteiro automático. Quando algo acontece em um sistema, ele entrega a notícia para o próximo na fila.
API
Tecnologia
Application Programming Interface. Como sistemas conversam entre si. O Painel usa a Polymarket API para buscar dados eleitorais automaticamente — sem abrir o site manualmente.Analogia: tomada elétrica. Você não precisa entender como a eletricidade funciona — só encaixa o plug.
Polymarket
Fonte de dados
Mercado de previsão. Pessoas apostam dinheiro real no resultado das eleições. O preço reflete a probabilidade percebida pelo mercado. Historicamente mais preciso que pesquisas porque quem aposta tem incentivo financeiro para acertar. $64.7M apostados nas eleições brasileiras de 2026.Analogia: bolsa de valores para eventos políticos. O "preço" de Lula a 43% significa que o mercado acha que ele tem 43% de chance de ganhar.
Funil
Marketing
A jornada desde conhecer o produto até virar assinante. Vê o post → faz o quiz → entra no trial → vira assinante. Começa largo (muita gente vê) e termina estreito (poucos pagam).Analogia: processo de compra em uma loja — cada etapa elimina quem não quer, até sobrar quem vai pagar.
Cron Job
Tecnologia
Tarefa automática em horário programado. O briefing das 7h é um cron job — todo dia, no horário certo, o sistema acorda e executa. Você não faz nada.Analogia: despertador automático para o sistema. Todo dia às 7h ele "acorda" e executa a tarefa.
Deploy
Tecnologia
Publicar o site na internet. Com a Vercel (que você já conhece), é automático — você faz push para o GitHub e o site atualiza em segundos.Analogia: imprimir e distribuir um livro. O código é o manuscrito — o deploy é quando chega nas livrarias.
FOMO
Psicologia de vendas
Fear Of Missing Out — medo de ficar de fora. "Últimas vagas no preço de fundador." As pessoas têm mais medo de perder do que vontade de ganhar.Analogia: contador regressivo da Black Friday. Não importa se você precisava — a sensação de acabar logo faz comprar.
Vibe Coding

Você descreve em português o que quer — "quero uma landing page dark com esfera 3D" — e a IA gera o código. Sem saber programar. É o que você vai fazer com Claude Code e Antigravity.Analogia: pedir para um arquiteto "quero algo moderno com muito vidro". Ele interpreta e constrói. Você só descreve o vibe.

O conjunto de ferramentas do produto. Stack do PULSE.JOURNAL: Pepper + Z-API + Make + Supabase + Vercel + Claude API + Beehiiv + Polymarket API + TSE API.Analogia: lista de ingredientes de uma receita. Cada um tem função — sem um deles o prato não fica igual.
